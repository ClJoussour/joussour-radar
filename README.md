# 📡 رادار جسور (Joussour Radar)

![Version](https://img.shields.io/badge/version-1.0.0-blue.svg)
![Status](https://img.shields.io/badge/status-active-success.svg)
![License](https://img.shields.io/badge/license-MIT-green.svg)

**رادار جسور** هو نظام أتمتة متكامل (Bot) تم تطويره لصالح نادي جسور. يهدف النظام إلى المراقبة الآلية واللحظية لإعلانات "مناقشات الدكتوراه" المنشورة على صفحة كلية العلوم الإسلامية عبر فيسبوك، وإعادة بثها فوراً إلى مجموعة (خلية الإعلام) على منصة تيليغرام مع نظام أزرار تفاعلي لإدارة المتطوعين للتصوير وتذكيرهم.

---

## ✨ المميزات (Features)

- **استخراج البيانات الآلي (Scraping):** مراقبة صفحة الكلية بدقة وسحب المنشورات الجديدة فور نزولها متجاوزاً حمايات فيسبوك.
- **التشغيل السحابي المجاني (Zero-Cost Serverless):** يعمل تلقائياً كل 60 دقيقة باستخدام `GitHub Actions` بدون الحاجة لسيرفر محلي أو مدفوع.
- **إدارة المتطوعين اللحظية (Interactive Webhook):** 
  - أزرار تفاعلية أسفل كل منشور (`أتطوع للتصوير`).
  - نظام تأكيد المهام (`هل قمت ببرمجة التذكير؟`).
  - معالجة سريعة جداً (بالتوازي) وحماية من التكرار عبر `Google Apps Script`.
- **الذاكرة الذكية:** تخزين أرقام المنشورات في ملف `last_posts_memory.json` لضمان عدم تكرار إرسال نفس الإعلان.

---

## 🏗️ المعمارية التقنية (Architecture)

يعتمد النظام على ثلاث ركائز تقنية رئيسية:
1. **Apify (The Engine):** يعمل كـ Proxy وسيط لاستخراج المنشورات من فيسبوك وتوفيرها بصيغة JSON.
2. **GitHub Actions (The Cron Job):** يشغل ملف `radar.py` كل ساعة لقراءة البيانات من Apify، مقارنتها بالذاكرة، وإرسال الجديد إلى تيليغرام.
3. **Google Apps Script (The Webhook):** يستقبل ضغطات الأزرار من تيليغرام، يعالج المنطق (تسجيل اسم المتطوع، إرسال مهام التذكير)، ويحدّث واجهة الرسالة فورياً.

---

## ⚙️ متغيرات البيئة (Environment Variables)

لكي يعمل النظام بشكل صحيح، يجب إعداد المتغيرات السرية التالية (Secrets) في إعدادات المستودع `Settings > Secrets and variables > Actions`:

| اسم المتغير | الوظيفة |
| --- | --- |
| `TELEGRAM_BOT_TOKEN` | التوكن الخاص بالبوت المستخرج من BotFather |
| `TELEGRAM_CHAT_ID` | المعرف الرقمي لمجموعة خلية الإعلام (يبدأ بـ `-100`) |
| `APIFY_API_TOKEN` | التوكن الخاص بحساب Apify للاتصال بواجهة API |

---

## 📂 بنية الملفات (File Structure)

- `.github/workflows/main.yml`: أوامر التشغيل الآلي كل ساعة.
- `radar.py`: الكود الأساسي المسؤول عن جلب البيانات وإرسال الإشعارات.
- `webhook_handler.gs`: (نسخة احتياطية) الكود المستضاف على سيرفرات جوجل لمعالجة الأزرار.
- `last_posts_memory.json`: قاعدة البيانات المصغرة لحفظ المنشورات المعالجة.
- `requirements.txt`: المكتبات المعتمدة (Requests).

---

## 📝 ملاحظات الصيانة

- في حال تغيير مجموعة تيليغرام مستقبلاً، يجب تحديث `TELEGRAM_CHAT_ID` في قسم الـ Secrets.
- في حال توقف الردود التفاعلية للأزرار، يرجى التأكد من أن رابط الـ Webhook الخاص بـ Google Apps Script لا يزال مرتبطاً بالبوت.
